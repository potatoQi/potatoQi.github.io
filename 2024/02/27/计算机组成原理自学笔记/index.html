指令集体系结构(ISA)

  • Instruction Set Architecture，简称ISA
  • 定义：指令集体系结构决定了计算机能够理解和执行哪些基本命令。简单来说，它就像是计算机的语言和字典，定义了计算机可以执行的所有操作，比如加法、减法、数据移动等。
  • ISA组成成分
    1. 指令集：这是一组命令，CPU可以直接理解和执行这些命令。每个指令执行特定的操作，例如算术运算、数据传输等。
    2. 数据类型：定义了CPU可以处理的数据种类，比如整数、浮点数等。
    3. 寄存器：这是CPU内部的小容量存储区域，用于快速访问数据。ISA 定义了有哪些寄存器以及它们的用途。
    4. 地址模式：决定了如何访问和存储内存中的数据，比如直接访问或间接访问。
    5. 存储器架构：涉及到数据如何在内存和其他存储设备中组织和访问。
  • 主要的ISA类型分为精简指令集计算机（RISC）和复杂指令集计算机（CISC）。RISC 架构的指令简单、统一，每条指令的执行时间相同，便于快速执行。CISC 架构的指令复杂、多样，一条指令可以执行多步操作。
  • MIPS 和 RISC-V 都是基于精简指令集计算机（RISC）原理设计的指令集体系结构。（这学期我们用RISC-V，因为它开源，未来中国很多企业也将会采用RISC-V）

计算机算术逻辑单元(ALU)

  • 定义：计算机的算术逻辑单元（Arithmetic Logic Unit，简称ALU）是CPU内部的一个关键组成部分，负责执行所有的算术和逻辑运算。这些运算包括基本的加法、减法、乘法、除法等算术运算，以及与（AND）、或（OR）、非（NOT）、异或（XOR）等逻辑运算。简单来说，ALU是计算机执行数学计算的“大脑”。
  • 工作原理：ALU根据来自CPU的控制信号执行操作。这些控制信号指定了要执行的具体操作（如加法或逻辑与）以及操作的输入数据（通常存储在寄存器中）。执行操作后，ALU会产生结果，并将其存回寄存器或内存中。此外，ALU还会更新CPU的状态标志（如零标志、进位标志、溢出标志等），这些标志用于指示操作的特定结果，如运算是否产生进位或溢出。

数据通路

  • 定义：数据通路是指在CPU内部，用于在各个组件之间传输数据的路径。这包括从寄存器到ALU、从内存到寄存器等的数据移动。数据通路的设计直接影响到CPU的性能，因为它决定了数据在CPU内部的移动速度和效率。

内存层次结构

  • 寄存器：位于CPU内部，提供最快的数据访问速度，但数量有限。

  • 主内存（RAM）：直接与CPU交互，速度较快，但成本高于辅助存储。

  • 缓存（Cache）：通常分为L1、L2和L3缓存，位于CPU和主内存之间，用于减少访问主内存的延迟。

  • 辅助存储：包括硬盘驱动器（HDD）、固态驱动器（SSD）等，提供大量的存储空间，但访问速度较慢。

输入/输出（I/O）

  • 输入/输出系统是计算机与外部世界（包括用户和其他计算机系统）通信的机制。I/O系统包括一系列的输入设备（如键盘、鼠标、摄像头）和输出设备（如显示器、打印机、扬声器）。此外，它还包括用于数据传输的接口和通道，如USB端口、网络接口等。
